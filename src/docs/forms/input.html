The HTML input element is the primary mechanism for HTML form input fields.

Input types

The input element has a type attribute and the default value is type="text", but various other types are available. The following examples show all the available types.

Input validation

When validating input fields, it is useful to show whether the entered input is valid or invalid. When doing so it is important to also make this clear for accessibility users using the aria-invalid attribute.

Constellation CSS makes use of the aria-invalid attribute to apply either an --success or --error color to mark the field visually. For invalid fields, any sibling element immediately following the input element will have smaller text in the --error color.

Fieldset

A fieldset is used to group a number of form fields that are logocally grouped. It is often used to group a number od checkboxes or radio buttons.

A fieldset is created using the fieldset element with a nested legend element and the nested content.

Fieldsets can also be marked as valid or invalid using the aria-invalid attribute.

Checkboxes

A checkbox is created using the <input type="checkbox"> element. Checkboxes are often grouped inside a fieldset to provide additional context.

Checkboxes

Radio button

A radio button is created using the <input type="cradio"> element. Radio buttons are often grouped inside a fieldset to provide additional context.

Radio buttons

Range input

The <input type="range"> does not visually look like a text input field and is thus styled differently.

Color input

The <input type="color"> is used to select a color using a color picker provided by the browser.

File input

The <input type="file"> is used to select one or more files to upload.
