Constellation CSS is a starter for building professional looking web experiences by styling semantic HTML elements.

It offers powerful customization options and CSS classes that assist with layout and styling by using well known CSS patterns.

It is used for building Hypermedia-Driven Applications and static websites such as blogs.

Why Constellation CSS?

Over the years, I have often searched for a CSS framework that gives me a good starting point for building professional looking web applications and websites. I do not want to go through the effort of styling html input elements from scratch every time I start a new project.

I have found a number of very promising options along the way, but when I want to do more complex custom styling I found that these solutions sometimes got in the way.

In recent years I have gathered alot of CSS knowledge and I finally decided to build my own CSS framework.

Inspiration

There are a number of solutions in the CSS space that have inspired some of the ideas in Constellation CSS. I would like to give credit to these projects that inspired me. Having a look at these may also give you a better idea of what Constellation CSS is about and how it compares to other solutions in the CSS space.

If I have seen further it is by standing on the shoulders of Giants.

Pico CSS

Pico CSS is a minimal CSS framework that styles semantic HTML elements. It is a great starting point for building simple websites and applications. I have used Pico CSS in the past and it has been a great experience. It is an excellent example of how powerful styling semantic HTML elements can be.

I have found that it can go too far in some areas by applying specific layouts to semantic elements. The <nav> is a good example of it. It creates a beautiful website header with a navigation bar, but when I wanted something different I have found it too complex to customize.

It's colors give you a palette of 20 colors and 19 shades for each color. This is great, but as a person with a limited feel for what looks good, I have no idea how many colors to use and which colors and shades work together.

Open Props

Open Props is a collection of CSS custom properties that can be used as a design system. It offers a wide range of properties for colors, typography, spacing, and more. I have used Open Props in the past and it has been a great experience. It is a great example of how powerful CSS custom properties can be for building a design system.

It has separate stylesheets for normalize and buttons, that do styling of semantic elements, but I still find myself drawn back to Pico CSS.

Cube CSS

CUBE CSS is a CSS methodology that's orientated towards simplicity, pragmatism and consistency. It's designed to work with the medium that you're working in (the browser) rather than against it.

It teaches you how to write better CSS.

Every Layout

Every Layout is a book that teaches you the "rudiments" of layout in CSS. It teaches you 13 reusable layout patterns that you can use in your projects. It is a great resource for learning about layout in CSS.

Constellation CSS has implemented most of these layout patterns as CSS classes to have them at your fingertips, but reading the book is recommended to understand the patterns and how to use them effectively.

Utopia

Utopia is a CSS utility for generating a fluid typography and spacing system. It is a great example of how to use CSS custom properties to create a responsive design system without the need for @media queries.

I have used Utopia as the basis for the typography and spacing system in Constellation CSS.

Tailwind CSS

Tailwind CSS is a utility-first CSS framework that provides a wide range of utility classes for styling your HTML. It is a great example of how to use utility classes to quickly style your HTML without the need for writing custom CSS.

Tailwind is by far the most famous and popular solution in this list. While I am not a fan of the utility-first approach, I do believe it is a great example of the power of utility classes. It tends to bloat your HTML with classes and fights against the cascade of CSS.

Daisy UI

Daisy UI is a component library built on top of Tailwind CSS that provides a wide range of pre-designed components for building your HTML. It is a great example of how to use pre-designed components to quickly build your HTML without the need for writing custom CSS.

Stellar CSS

Stellar CSS is a CSS framework that is built on top of the Datastar Hypermedia framework and Rocket web components.

Rocket and Stellar are part of the Datastar Pro offering, which requires a commercial license. I do not have a licence and thus I have never used either of these solutions, but I am an active member of the Datastar community and follower of the Youtube channel and Discord channel. I have even open-sourced my own Datastar SDK, which is an unofficial SDK for Java, Spring MVC and the JTE templating engine.

As you may have guessed, the name of Constellation CSS is inspired by Stellar CSS. When I first read about the development of Stellar CSS and what inspired it, I imaged what it could be. Constellation CSS is my attempt at building the CSS framework that I imagined when I read about Stellar CSS.

Using Constellation CSS

If the ideas of Constellation CSS resonate with you, you can use it in your projects by including the CSS file in your HTML. You can also customize the styles by overriding the CSS variables or by adding your own styles on top of it.
